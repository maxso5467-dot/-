# 开发日志

## 2026-07-17

### 微信小程序

- 新增 `miniprogram/` Taro 4.2 React微信小程序工程。
- 新增小程序账号登录和微信快捷登录入口。
- 新增健康首页，展示AI状态、档案指标和近期问诊。
- 新增智能问诊页，支持症状自查、普通/儿童/老人模式、中英双语和连续追问。
- 新增健康档案编辑和问诊历史页面。
- 新增 `/api/v1/auth/wechat/login`，通过服务端 `code2Session` 换取OpenID并签发项目JWT。
- 新增 `wechat_accounts` 表，安全关联微信OpenID与本地用户。
- 微信AppSecret和DeepSeek密钥均只保存在后端环境变量中。
- Taro微信小程序生产构建通过，五个页面产物完整生成。

### 参考项目融合改进

- 参考第二组方案增加儿童和老人专属问诊模式：儿童模式面向监护人追问年龄、体温、精神状态、进食饮水和呼吸情况；老人模式采用大字体、通俗短句并重点关注慢病、用药、跌倒和活动能力。
- 参考第二组实时翻译方案增加中文和中英双语回复选择，语言设置进入AI上下文。
- 参考第十五组症状自评向导增加部位、主要症状、持续时间、严重程度和伴随症状的分步选择，并自动生成规范问诊描述。
- 参考第十五组智能导诊方案强化结构化科室推荐，并在问诊报告中展示风险原因和行动建议。
- 增加问诊报告打印样式，可隐藏操作区后打印对话、症状、风险、科室和医生摘要。
- 问诊模式、语言和症状向导来源经过后端白名单校验并保存至消息上下文。

### 新增

- 增加结构化问诊数据表：症状提取、连续追问、问诊总结和医生复核。
- 接入 DeepSeek Chat Completions API，支持通过环境变量选择模型和服务地址。
- 文本问诊输出症状、身体部位、持续时间、严重程度、追问、风险等级、建议科室、行动建议和医生摘要。
- 增加连续追问机制，自动关联用户回答并更新 `pending`、`answered` 状态。
- 增加问诊进度展示，历史会话可恢复追问状态。
- 增加图片分析状态分类：`UNSUPPORTED`、`NOT_CONFIGURED`、`NETWORK_ERROR`、`MODEL_ERROR` 和 `INVALID_IMAGE`。
- 增加结构化问诊结果和图片状态前端组件。

### 修复

- 修复“没有呼吸困难”被关键词规则误判为紧急风险的问题。
- 修复中文时长“两天”等表达无法识别的问题。
- 修复“不能正常负重”等包含修饰词的高风险表达识别。
- 防止同一问诊字段生成重复的待回答问题。
- 增加会话归属校验，避免用户读取其他用户的问诊详情。

### 安全与降级

- DeepSeek 文本模型不支持图片分析时返回真实原因，不再使用网络异常作为占位提示。
- 未配置模型或外部模型调用失败时，返回明确的本地结构化规则结果。
- 规则引擎风险等级只能被模型提高，不能被模型降低。
- API Key 仅从 `.env` 读取，`.env` 已加入 Git 忽略列表。
